# 费曼学习法 Audio2Audio 应用设计文档

## 1. 产品概述

一款基于 Qwen3.5-Omni 实时语音大模型的费曼学习法应用。用户通过口头讲解题目，AI 以"学生"身份倾听、追问、复述，直到被用户真正讲懂。

核心目标：验证"费曼学习法 + 实时语音 Agent"是否可以成立为一个独立学习产品方向。

## 2. 技术栈

| 层级 | 技术选型 |
|------|---------|
| 前端框架 | React + Vite + TypeScript |
| UI 方案 | Tailwind CSS + shadcn/ui |
| 状态管理 | Zustand |
| 音频处理 | Web Audio API + 浏览器麦克风 |
| 后端 | Node.js + Express + ws（WebSocket 代理） |
| AI 模型 | Qwen3.5-Omni-Flash-Realtime |

## 4. 前端核心组件

| 组件 | 职责 |
|------|------|
| `HomePage` | 首页：产品说明 + 入口按钮 |
| `CreateTaskPage` | 创建任务：题目输入 + 学科选择 |
| `SessionPage` | 核心讲解页：音频交互 + 状态展示 |
| `ResultPage` | 结果页：讲懂判定结果 + 反馈 |
| `AudioRecorder` | 麦克风采集 + 音频流发送 |
| `AudioPlayer` | AI 音频流播放 + 中断处理 |
| `UnderstandingTracker` | 理解进度展示（已理解/未理解点） |
| `useSessionStore` | Zustand Store：管理对话状态机 |

## 5. 后端核心模块

| 模块 | 职责 |
|------|------|
| `ws-server` | WebSocket 服务，端到端透传音频 |
| `proxy-handler` | 请求转发，API Key 管理 |
| `understanding-judge` | 讲懂判定逻辑（基于 Qwen 回复判断） |
| `session-manager` | 会话状态管理 |

## 6. 对话状态机（S0-S7）

```
S0 待开始 ──▶ S1 AI开场 ──▶ S2 用户讲解中 ──▶ S3 AI等待追问时机
                                    │                    │
                                    ▼                    ▼
                              S7 任务中止          S4 AI追问/反馈
                                                        │
                                                        ▼
                                                  S5 接近讲懂
                                                        │
                                                        ▼
                                                  S6 已讲懂 ──▶ 结果页
```

### 状态说明

- **S0 待开始**：用户创建任务后，等待点击"开始讲解"
- **S1 AI开场**：AI 用一句话建立角色和任务感，如"你来讲给我听吧"
- **S2 用户讲解中**：用户持续讲解，系统实时收音、转写、积累语义
- **S3 AI等待追问时机**：用户刚说完，评估是否需要回应
- **S4 AI追问/反馈**：AI 进行追问/澄清/阶段确认/复述总结
- **S5 接近讲懂**：理解度已较高，还差 1-2 个关键点
- **S6 已讲懂**：系统确认 AI 被讲懂，进入结果页
- **S7 任务中止**：用户主动退出/网络异常/长时间沉默

## 7. MVP 阶段功能范围

### P0 必做

- 文本题目输入
- 实时语音讲解（按住说/点击开始）
- AI 语音追问
- 转写可视化
- 已理解/未理解点展示
- 讲懂判定
- 结果页

### 暂时不做

- OCR 识题
- 多角色 AI
- 情绪陪伴路线
- 社交分享
- 多学科复杂 rubric
- 很强的中途插话打断

## 8. 关键交互流程

1. **创建任务** → 输入题目，存入 Zustand
2. **开始讲解** → 连接后端 WS，前端采集麦克风音频
3. **语音发送** → 音频流实时发送给后端，后端透传给 Qwen
4. **AI 回应** → Qwen 返回音频+文字，后端透传给前端
5. **理解进度** → 后端解析 AI 回复，判断理解度，更新前端展示
6. **追问循环** → 重复步骤 3-5，直到 S6 讲懂
7. **结果页** → 展示讲懂判定、复述、反馈

## 9. 错误处理

| 场景 | 处理方式 |
|------|---------|
| 网络断开 | 弹出提示，暂停录音，可重连继续 |
| Qwen API 异常 | 后端捕获错误，返回友好消息给前端 |
| 麦克风权限拒绝 | 引导用户开启权限 |
| 长时间无响应 | 超时提醒，可选择继续或结束 |

## 11. 验收标准

- 用户可以输入题目并开始讲解
- AI 可以实时回应语音追问
- 界面显示已理解/未理解点
- AI 能判断是否被讲懂并展示结果页
- 整体延迟在可接受范围内（<3秒）
